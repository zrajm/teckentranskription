How to Generate the Data File for the Lexicon
=============================================

Fetch HTML from teckensprakslexikon.su.se
-----------------------------------------
Download all the lexicon definitions, from Svenskt Teckenspr책kslexikon using
the following commands. (These commands below will download tens of thousands
of HTML files, so you might want make sure to run them in a new, clean
directory.) :)

    seq -f http://teckensprakslexikon.su.se/ord/%05g 001 17248 >WORDS.url
    wget -ci WORDS.url

To determine which large number to use visit the dictionary manually with your
browser, and experiment a bit.


ID-to-Word Mapping File
-----------------------
Use the `lexiconhtml2words` command to generate a lexicon-ID-to-word-mapping
file from the downloaded HTML files, like this:

    bin/lexiconhtml2words DOWNLOAD_DIR >svtsplex-id2sv-2.txt


Teckenspr책kslexikon Transcription Dump File
-------------------------------------------
This file is called `svtsplex-export.tab` in this repo, it contains the sign
language transcriptions for each sign in the lexicon in an (almost)
human-readable format.

The file itself is a database export, which can be requested from the office of
Svenskt teckenspr책kslexikon. It has a peculiar format, using carriage return
(ASCII \x0d or \r) to delimit records, and vertical tabs (ASCII \x0b or \v) do
delimit the fields within a record.

Each record starts with the five-digit lexicon ID number of the sign in
question, followed by a tab, and then the data fields (with numbers) in each
field. An empty field contains '---'. It looks somewhat like this (except
without newlines).

    00003\t  1.---\v2.hj채ssan\v3.---\v4.---\v5.---\v[...]\v105.---\r
    00004\t  1.---\v[...]


Generate JSON
-------------
The JSON is generated from ID-to-translation mapping file + the transcription
dump file, like this:

    bin/mklexicon --webapp \
        --images=STUFF/teckensprakslexikon.su.se-2017-12-01/IMAGES \
        --words=svtsplex-id2sv.txt <svtsplex-export.tab >lexicon-data.js


[eof]
